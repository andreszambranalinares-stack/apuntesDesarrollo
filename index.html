Roles del Equipo (Definiciones Completas)

Analista de Sistemas

Realiza un estudio del sistema para dirigir el proyecto. Garantiza las expectativas del cliente. Elabora el ERS.

Analista Programador

"Desarrollador". Visión amplia. Diseña soluciones amigables para la codificación y participa activamente en ella.

Diseñador

Evolución del analista. Realiza el diseño de la solución (diagramas, estructura) basada en el análisis previo.

Arquitecto de Software

La "argamasa" que cohesiona. Conoce frameworks y patrones. Revisa que se usen los recursos apropiados.

Diagramas UML (Unified Modeling Language)

Lenguaje visual estándar para especificar, construir y documentar artefactos de software.

1. Diagrama de Clases (Estructural)

Describe la estructura estática del sistema. Muestra las clases, sus atributos (datos), métodos (acciones) y las relaciones entre objetos (asociación, herencia, composición).

Se utiliza principalmente en la fase de Diseño para definir la arquitectura de datos.

2. Diagrama de Secuencia (Comportamiento)

Muestra la interacción de un conjunto de objetos a través del tiempo. Se centra en el orden cronológico de los mensajes que se intercambian. Utiliza "líneas de vida" verticales.

Se utiliza para modelar la lógica de un caso de uso específico.

3. Diagrama de Actividad (Comportamiento)

Representa el flujo de trabajo paso a paso de negocio y operacionales de los componentes de un sistema. Muestra el flujo de control general.

Es similar a un diagrama de flujo tradicional (Inicio -> Proceso -> Decisión -> Fin).

Glosario POO (Definiciones Completas)

Clase

Es un tipo de dato definido por el programador. Describe propiedades y métodos comunes para los objetos creados a partir de ella. No se usa directamente: debe instanciarse.

Objeto (Instancia)

Elemento creado a partir de una clase. Tiene sus propios valores en memoria.

Variable (Referencia)

Identificador que almacena la dirección de memoria de un objeto; permite operar sobre él. La variable no es el objeto.

Instanciar

Crear un objeto a partir de una clase usando new.

Constructor

Método que se ejecuta al instanciar un objeto y sirve para inicializar sus propiedades.

Miembro

Cada propiedad o método definido dentro de una clase.

Encapsulación

Protege los datos internos del objeto controlando el acceso mediante modificadores y métodos específicos.

Composición

Una clase contiene objetos de otras clases como propiedades. Al crear un objeto, también se crean en cadena los objetos que contiene.

Herencia

Una clase recibe propiedades y métodos de otra. Normalmente es simple. Puede sobrescribir métodos heredados.

Polimorfismo

Permite que un mismo método o referencia opere con objetos de distintos tipos derivados, adaptando el comportamiento.

Clase Abstracta

Clase que no puede instanciarse. Puede contener métodos implementados o abstractos. Obliga a implementar métodos en clases hijas.

Interfaz / Protocolo

Contrato que define métodos que una clase debe implementar, sin implementación concreta.

Paso de mensaje

Los objetos se comunican llamando métodos entre sí, solicitando acciones.
